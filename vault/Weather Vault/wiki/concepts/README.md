# Concepts

Ideas and methods that matter to the project. HRRR, MOS (Model Output Statistics), TAF, calibration, Kelly sizing, ensemble spread, per-airport bias correction, temporal leakage, etc.

## Conventions

- **Filename**: the concept's canonical name (e.g. `HRRR.md`, `Model Output Statistics.md`, `Kelly Sizing.md`, `Calibration.md`).
- **Frontmatter**:
  ```yaml
  ---
  tags: [concept, <domain>]
  type: concept
  related: [[...]]
  ---
  ```
- Each page should have:
  - **Definition** — what the concept is, in 1–3 sentences
  - **How it applies to this project specifically** — not a general textbook definition; the project-relevant angle
  - **Sources** — wiki pages and raw sources that discuss it
  - **Related concepts** with wikilinks

- Use `[[wikilinks]]` aggressively.
- `vault-scribe` maintains these automatically on ingest.
